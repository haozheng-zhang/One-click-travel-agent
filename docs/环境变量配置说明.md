# 环境变量配置说明

## 概述

本项目使用环境变量来管理敏感信息（如 API Key），以保护代码库安全。

## 文件说明

### 1. `.env.example` ✅ 提交到 GitHub
- **用途**: 环境变量模板，展示需要配置哪些变量
- **内容**: 示例值或空值
- **行为**: 可以安全地提交到 GitHub

### 2. `.env` ⚠️ 不提交到 GitHub
- **用途**: 开发环境配置
- **内容**: 已替换为示例值 `your_deepseek_api_key_here`
- **行为**: 被 `.gitignore` 忽略，不会提交到 GitHub

### 3. `.env.local` 🔒 本地文件，不提交到 GitHub
- **用途**: 存储本地机器上的实际 API Key
- **内容**: 你的真实 Deepseek API Key
- **行为**: 被 `.gitignore` 忽略，仅在本地使用

## 工作流程

```
.env.local (优先，包含真实 API Key)
    ↓ (如果存在，则使用)
.env (备选，示例值)
```

## 使用步骤

### 首次设置

1. **复制模板**
   ```bash
   cp backend/.env.example backend/.env
   ```

2. **创建本地配置文件**
   ```bash
   cd backend
   ```
   创建文件 `.env.local`，内容如下：
   ```
   DEEPSEEK_API_KEY=sk-your-actual-api-key-here
   ```

3. **获取 API Key**
   - 访问 https://platform.deepseek.com
   - 创建账户或登录
   - 在 API Keys 页面生成你的 API Key
   - 将 API Key 复制到 `.env.local` 中

4. **验证配置**
   ```bash
   python main.py
   ```
   启动日志应该显示：`✓ Deepseek API Key 已配置`

### 生产环境

#### 方案一：使用环境变量（推荐）
```bash
# 设置环境变量
export DEEPSEEK_API_KEY="your_api_key_here"
python main.py
```

#### 方案二：使用 `.env.local`
在服务器上创建 `.env.local` 文件，包含实际的 API Key。

## 提交到 GitHub

只需要提交以下文件：
- ✅ `.env.example` - 配置模板
- ✅ 其他所有代码文件

不要提交：
- ❌ `.env` - 被 .gitignore 忽略
- ❌ `.env.local` - 被 .gitignore 忽略

## 常见问题

**Q: 为什么收到警告 "Deepseek API Key 未配置"？**

A: 原因可能是：
1. 没有创建 `.env.local` 文件
2. `.env.local` 中的 API Key 值为空
3. 环境变量未正确设置

**Q: 如何切换 API Key？**

A: 编辑 `.env.local` 文件，更新 `DEEPSEEK_API_KEY` 的值。

**Q: 可以重命名 `.env.local` 吗？**

A: 不推荐。如需自定义，请编辑 `main.py` 中的 `env_local_path` 变量。

## 安全最佳实践

1. ✅ 永远不要把真实的 API Key 提交到 Git
2. ✅ 使用 `.env.local` 或环境变量存储敏感信息
3. ✅ 定期轮换 API Key
4. ✅ 检查 `.gitignore` 确保敏感文件被忽略
5. ✅ 团队成员应该各自创建自己的 `.env.local`
